This is what my friend, and oft critic of the models, Bill Gray is fond of asking seminar speakers and graduate students and I suppose you too are wondering what this is all about. In many ways the PCMDI WXMAP Web is "yet another weather map page" and even I will toss in some of my favorite weather sites for good measure. However, I see my Web as having two distinctive features:

  • A Stong Orientation toward Direct Model Intercomparison
  • Model Verification using Statistics and a Map/Web Archive
While many weather pages have satellite and radar images and display current weather conditions, we only offer NWP maps. So now you are wondering, "what's an NWP?"

NWP stands for "Numerical Weather Prediction," i.e., the "computer models" you've heard about (and Bill Gray disapproves of becoming too enamored with). NWP is actually a bit of a misnomer as the computer models do not directly forecast the weather as most people think of it ("is it going to rain, Mike?"). Rather, meteorologists make "weather maps" from the output of the NWP model forecasts (e.g., winds and temperature) and/or diagnosistics (e.g., rainfall) and then infer the weather from the maps.

These pages provide basic weather maps using data from the two leading NWP agencies in the United States -- the National Centers for Environmental Prediction (NCEP) and the Navy's Fleet Numerical Meteorology and Oceanography Center (FNMOC). We use two centers because PCMDI believes model intercomparison is the key to model improvement and this web is arranged accordingly. Further, we focus on global models because PCMDI's mission is to promote global climate model improvement through intercomparison and diagnosis. The two models of interest are: 1) the NCEP Medium Range Forecast (MRF) model -- both the short range (0-72 h at 00, 06, 12 and 18 UTC) "Aviation" (AVN) run and the longer range (0-388 h at 00 UTC) MRF runs; and 2) the FNMOC Navy Operational Global Atmospheric Prediction System (NOGAPS or NGP in these pages).

PCMDI's interest in NWP stems primarily from the use of NWP data in climate model verification. Of special interest is atmospheric reanalysis where the observations (what Bill Gray would call "real" data) are "put through an unchanging analysis/forecast assimilation wringer" to produce long times series (greater than 10 years) suitable for climate applications. The NCEP/NCAR project is the biggest and prototypical. More information on reanalysis will be coming out on PCMDI's web in the next few months.

Web Structure

The root of the web is:

http://www-pcmdi.llnl.gov/fiorino/wxmap

If you click on this base, you'll see a lot of files. All my html documents end in the with .htm extension, but you'll also find .pl files (perl scripts which actually generate the web) and .sh Bourne shell scripts I still use for some management functions. Here are the key files:

wx.htm The current main page
wx.conus.htm The starting page for the CONUS area, use this link if you want to go to this area
wx.asia.htm The starting page for the East Asia area, this link is for people who are only interested in this region
avn.conus.htm The Map page for the AVN model in CONUS, use this if you want to always go to this area and model
mrf.conus.htm The current page for the MRF model in CONUS
ngp.conus.htm The current page for the NOGAPS model in CONUS
avn.europe.htm The Map page for the AVN model in Europe
mrf.europe.htm The current page for the MRF model in Europe
ngp.europe.htm The current page for the NOGAPS model in Europe
wx.YYMMDDHH.htm Home page for the web at the YYMMDDHH "Date Time Group"
wx.AREA.YYMMDDHH.htm The main page for the selecting models the AREA area at the YYMMDDHH "DTG". Currently, AREA="conus" or "asia"
MODEL.AREA.YYMMDDHH.htm The main page for the maps in the AREA area for model MODEL at the YYMMDDHH "DTG". Currently, AREA="conus" or "asia" and MODEL="avn" or "mrf" or "ngp"
